## Описание задачи

Задача демонстрирует обработку исключений при параллельном выполнении асинхронных операций в C#. В реальных приложениях часто возникает необходимость выполнять несколько асинхронных операций одновременно и корректно обрабатывать возможные ошибки в каждой из них.

## Используемые технологии и библиотеки
### 1. .NET Core / .NET 5+

- Асинхронное программирование с использованием ```async/await```

- Пространства имен:

    - ```System``` - базовые типы и классы

    - ```System.Threading.Tasks``` - для работы с задачами (Task)

### 2. Ключевые концепции C#

- **Async/await** - для асинхронного программирования

- **Task Parallel Library (TPL)** - для управления параллельными задачами

- **Исключения** - механизм обработки ошибок

- **```Task.WhenAll()```** - ожидание завершения всех задач


## Детали реализации
### Структура программы

- ```  static async Task Main() ``` <- Асинхронная точка входа
    1. Создание массива задач
    2. Параллельный запуск
    3. Обработка результатов и ошибок

- ``` static async Task<int> GetDataAsync(int taskNumber) ```
    1. Симуляция асинхронной операции
    2. Случайное выбрасывание исключений
    3. Возврат результата

### Логика работы метода GetDataAsync

1. **Имитация асинхронной операции**: ```await Task.Delay(100)``` - задержка 100 мс

2. **Условное выбрасывание исключения**: задачи с четными номерами генерируют ошибку

3. **Возврат результата**: задачи с нечетными номерами возвращают ```taskNumber * 10```

### Обработка исключений

Особенность решения - использование двух подходов:

1. **Обработка через ```Task.WhenAll()```:**

    - При вызове await ```Task.WhenAll(tasks)``` выбрасывается первое исключение

    - Но остальные задачи продолжают выполняться

2. **Анализ состояния каждой задачи**:

    - ```IsFaulted``` - проверка на наличие исключения

    - ```IsCompleted``` - проверка на успешное завершение

    - Доступ к результату через ```Result``` свойство

    - Доступ к исключению через ```Exception.InnerException```
